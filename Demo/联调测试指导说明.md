# Android 模拟器 + ROS2 机械臂联调测试指导

本文档用于指导你在 **Android 虚拟环境（Emulator）** 与 **ROS2 机械臂侧（真机或仿真）** 间验证当前 SDK 的三类能力：

1. MoveJ 指令下发（`/arm/movej`）
2. 夹爪指令下发（`/arm/gripper`）
3. `joint_states` 状态反馈订阅（`/joint_states`）

---

## 1. 环境准备

### 1.1 ROS2 侧

确保已安装并启动：

- `rosbridge_server`
- 机械臂控制节点（订阅 `/arm/movej`、`/arm/gripper`）
- 机械臂状态发布节点（发布 `/joint_states`）

启动 rosbridge：

```bash
ros2 launch rosbridge_server rosbridge_websocket_launch.xml
```

### 1.2 Android 侧

- 使用 Android Studio 启动一个 Emulator（推荐 API 30+）。
- **重点**：模拟器访问宿主机端口使用 `10.0.2.2`，不是 `localhost`。
- 因此 WebSocket 地址应配置为：

```text
ws://10.0.2.2:9090
```

---

## 2. ROS2 侧联调观测命令

建议开 3 个终端窗口：

```bash
ros2 topic echo /arm/movej
```

```bash
ros2 topic echo /arm/gripper
```

```bash
ros2 topic echo /joint_states
```

如果你需要先验证是否存在这些话题：

```bash
ros2 topic list
```

查看消息类型：

```bash
ros2 topic info /arm/movej
ros2 topic info /arm/gripper
ros2 topic info /joint_states
```

---

## 3. Demo 操作步骤

1. 安装并运行 `Demo/app/src/main/java/com/example/robotarm/demo/RobotArmDebugActivity.kt`。
2. 点击 **Connect** 建立 WebSocket 连接。
3. 点击 **MoveJ Sample**，检查 ROS2 终端 `/arm/movej` 是否收到 `Float64MultiArray`（6 位 `data`）。
4. 点击 **Gripper Sample**，检查 `/arm/gripper` 是否收到 `Float64MultiArray`（3 位 `data`）。
5. 点击 **Subscribe JointStates**，观察页面中状态文本是否持续更新。

---

## 4. 通过判定标准

- Connect 后无异常，日志显示连接动作成功。
- `/arm/movej` 正确收到长度为 6 的 `data`。
- `/arm/gripper` 正确收到长度为 3 的 `data`。
- 页面能持续接收到 `joint_states` 并显示 `name / position / velocity / effort`。

---

## 5. 常见问题排查

### 5.1 连不上 rosbridge

- 检查 rosbridge 是否启动。
- 检查端口是否为 9090。
- 检查地址是否误填为 `localhost`（应为 `10.0.2.2`）。
- 检查主机防火墙。

### 5.2 能发命令但机械臂不动

- 机械臂控制节点是否订阅了正确话题。
- 是否要求 `std_msgs/msg/Float64MultiArray`。
- 数据长度是否符合约定（MoveJ=6，Gripper=3）。

### 5.3 订阅不到 joint_states

- 机械臂侧是否持续发布 `/joint_states`。
- 发布类型是否为 `sensor_msgs/msg/JointState`。
- SDK 订阅 topic 名与实际一致（默认 `/joint_states`）。
