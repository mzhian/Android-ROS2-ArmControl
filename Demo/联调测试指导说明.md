# Android 模拟器 + ROS2 机械臂联调测试指导

本文档用于指导你在 **Android 虚拟环境（Emulator）** 与 **ROS2 机械臂侧（真机或仿真）** 间验证 SDK 的核心通讯与控制能力。

---

## 1. 环境准备

### 1.1 ROS2 侧 (服务端)
启动 `rosbridge_server`（WebSocket 默认端口 `9090`）：
```bash
ros2 launch rosbridge_server rosbridge_websocket_launch.xml
```

### 1.2 Android 侧 (客户端)
- 使用 Android Studio 启动 Emulator (推荐 API 30+)。
- **网络重点**：模拟器访问宿主机使用 `10.0.2.2`。
- **配置地址**：在 Demo 界面或代码中确保指向 `ws://10.0.2.2:9090`。

---

## 2. 界面交互与效果判定

### 2.1 连接状态 (Status Bar)
界面下方会实时显示中文状态：
- **空闲** / **正在连接...**
- **已连接 (Connected)**：代表链路已打通。
- **尝试自动重连中...**：网络波动时 SDK 会自动尝试找回连接。

### 2.2 异常反馈 (Toast)
如果点击按钮时处于**未连接**状态，或者通讯过程中发生异常，屏幕下方会弹出 **Toast 提示**（如：“SDK 异常: WebSocket not connected”），开发者无需翻看日志即可感知错误。

---

## 3. 调试日志 (Logcat)

强烈建议在 Android Studio 的 Logcat 中建立以下过滤器：

| 标签 (Tag) | 监控级别 | 说明 |
| :--- | :--- | :--- |
| **RosbridgeClient** | `DEBUG` | **报文级日志**。查看发送(>>SEND)和接收(<<RECV)的原始 JSON。 |
| **RobotArmDemo** | `DEBUG` | **业务级日志**。记录用户点击行为及关节数据解析反馈。 |

---

## 4. 核心话题验证 (ROS2 侧)

在宿主机开启终端观测：
- 运动指令：`ros2 topic echo /arm/movej`
- 夹爪指令：`ros2 topic echo /arm/gripper`
- 状态反馈：`ros2 topic echo /joint_states`

---

## 5. 常见问题
- **无法连接**：确认 `10.0.2.2` 地址正确；确认宿主机 9090 端口未被防火墙拦截。
- **指令无效**：通过 `RosbridgeClient` 日志确认 JSON 格式是否符合 ROS2 节点的 `Float64MultiArray` 要求。
- **数据不更新**：确认机械臂侧是否在持续发布 `/joint_states` 话题。
